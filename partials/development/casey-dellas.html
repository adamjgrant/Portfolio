Fitness trainer and nutrional counselor Casey Dellas wanted to upgrade the experience of his website, pictured below:

Dellas's old site was plagued with poor design and navigation.

We set out to create a consistent look and feel for the site while making it easy for users to navigate on any device. The result is pictured below.

Here is a demonstration of the different behavior on a variety of device types:




Using jQuery UI, we were able to create an appearance similar to a native app.

I used Kickstrap 1 with jQuery mobile exclusively for mobile devices. Keeping the two sites separate is not noramlly something I do, but we had some stellar design tricks that really worked best on desktop. If taken away on mobile, it would require a lot of re-engineering for it to make sense.

User Testing

Before setting out too far, I did some user testing with my product, Lab95, on the current site to identify user's initial impressions.

I used my own product, Lab95 to test users' memories to Five Second Tests--showing the website only briefly.

Overall, the numbers weren't the worst I've seen, but there was certainly some confusion about what Casey was offering and whether he was a reputable source of business.

We agreed on some basic design improvements--Testimonials should be embedded into the content, not separated onto one page. There should only be submenus for content within a page. The home page should have a clear call to action with a secondary action.

To keep the navigation clear and consistent, I also wrote some custom code for desktop to fix headers above the page and glide content under a soft gradient as the user scrolled.


Content appears in a large, readable font and slides under the header of the currently viewed section.
