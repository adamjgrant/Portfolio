First Steps

If you're looking at this page from your local installation, you've successfully got Kickstrap up and running with the example pages. You should skip to Editing content. otherwise, continue reading.

Download Kickstrap

(Alpha: Not yet implemented)

To download Kickstrap, you'll need Node Package Manager. . If you already have this installed, run

sudo npm install -g kickstrap

Create a new Kickstrap site

  1. Navigate to the directory where you'd like to create a new Kickstrap site and run 
    kickstrap new . (Alpha: not yet implemented)
  2. If you want to make the site in a directory at the current location, let's say it's called "mysite" replace the "." with "mysite"
  3. kickstrap new mysite
  4. Whichever the case, make sure you are now in the site's directory.
  5. To run the example content created, run 
  6. kickstrap watch

    (Alpha: use roots watch)

    This is actually running the command roots watch and we've remapped to this from the kickstrap command for ease-of-use. That being said, all roots commands will work here as well.

  7. If everything worked correctly, you should now see our Kickstrap example page. If not, or something isn't right, file or look for an issue at the Kickstrap GitHub page.

Continue reading: Compiling in Roots to learn how best to compile, watch, and prepare for a production website.

Editing content

Also see:

  • Creating pages

In Roots, your pages are generated with the files in views/.

layout.jade and blank.jade are base files. By default, files will extend (add themselves to) blank.jade. This is the case for partials and layout.jade, each of which we'll explain next.

The pages in the example app extend layout.jade, which provides the repetitive header and footer content you'll use application-wide. In the case of this page, partials are used inside index.jade via angular. This allows a single page to refresh with content for only a small part of the page. By loading only the partial, all the head content and JavaScript dependencies do not need to be set up again.

Kickstrap uses both Angular partials and Jade partials. Angular partials load in real time, on request in the browser. They can be found in views/partials/.

Jade partials are compiled once in the project and added to the final .html files. These are found in views/ and should start with an underscore. (e.g. "_footer.jade")

Change theme and basic settings

We've attempted to group as much of the runtime settings into settings.js in the js directory.

(Alpha: Subject to change)

theme: 'kickstrap'

To change the theme, change the name to match a .css file in the themes directory. To use the default theme with bootstrap, just use "bootstrap".

Removing/Editing growl

You may have noticed a "growl" alert appears on the home page when you open Kickstrap. This is generated in index.jade towards the bottom.

:coffeescript
  k$.ready ->  
    $.growl
      title: 'Welcome to Kickstrap!'
      text: 'Check out the documentation to get started.'
      type: 'success'
      delay: 3000

k$.ready is a JavaScript function that will execute anything below it once JSPM has finished running its components. As such, any code that is dependent on Kickstrap apps or angular will load once everything is ready and available. This can be included in multiple places on one page.

Growls are a jQuery extension available as an app called Tiny Growl. This app must be included in k$.apps to run correctly. To remove the growl completely, simply delete $.growl and the indented code below it.

For more information on using growls, visit the GitHub documentation.

Editing the header

The header that provides the navigation bar is a Jade partial. This is automatically added to any view with the code: 

include _header
_header.jade can be found in views/.
