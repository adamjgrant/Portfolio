Casey's site was a fun project because I was able to start fresh and use modern web technologies.

Having run Casey's previous site through usability tests, we determined users did not find the design attractive or trustworthy, weren't even sure of what the site was about, and had a difficult time navigating overall.

To add to this trouble, the poor navigation caused much of the content to be hidden away rather than supporting Casey's expertise.


Navigation items lift and stack as the user scrolls down, much like a fixed-weight machine. The extra visual tricks took a lot of configuring and work, but in the end, the expansive and animated result was definitely worth it.

Casey and I decided on several large changes. First, the amount of content on the site didn't justify the extreme amount of subnavigation. We removed the page for testimonials and instead peppered these quotations in with the content across pages.

I also used several scrolling tricks with the desktop site to engage and entertain the reader without distracting her from the content.

Mobile




Using jQuery UI, we were able to create an appearance similar to a native app.

Casey's site was one of the few times I created a separate code base for the mobile styles and some of the markup. This was a decision I made halfway into writing CSS responsive rules because there was still a lot of behavior going on that I felt should only be present on the desktop.

Specifically, I wanted users on the Desktop to enjoy the benefits of parallax scrolling, while users on mobile may need to access information like pricing and contact details more quickly.

This also allowed me to play with the jQuery UI library and create more iOS-specific effects such as menus in landscape mode.
